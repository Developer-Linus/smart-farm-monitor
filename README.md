# Smart Farm Monitor

An end-to-end IoT system for tomato farmers that combines real-time sensor monitoring, AI-powered leaf disease detection, and automated irrigation control. The ESP32-CAM hardware sends data to a Flask backend, which runs a TensorFlow Lite model and serves a React dashboard.

## Features

- **Live sensor dashboard** - temperature, humidity, and soil moisture updated every 3 seconds
- **AI leaf disease detection** - MobileNetV2 model identifies tomato leaf conditions from ESP32-CAM images
- **Automated irrigation** - pump control based on configurable soil moisture thresholds
- **Manual irrigation override** - turn the pump on or off directly from the dashboard
- **Alert system** - instant warnings when sensor readings exceed safe thresholds
- **Sensor history charts** - interactive graphs showing trends over time
- **Agrovet finder** - locates the nearest agricultural supply stores using your GPS position
- **User authentication** - secure login and registration with hashed passwords

## System Architecture

```text
ESP32-CAM
  |-- DHT22 (temperature + humidity)          Every 5 seconds
  |-- Capacitive soil moisture sensor         Every 5 seconds
  |-- Camera (leaf image)                     Every 30 seconds
  |-- Relay (irrigation pump)                 Controlled by Flask response
        |
        | HTTP POST (JSON / JPEG)
        v
Flask Backend (Python)
  |-- Receives and stores sensor readings
  |-- Runs TFLite inference on leaf images
  |-- Manages irrigation state
  |-- Serves the React frontend (frontend/dist/)
        |
        | REST API  (/api/...)
        v
React Dashboard (browser)
  |-- Polls /api/status  every  3 seconds
  |-- Polls /api/history every 10 seconds
  |-- Displays charts, alerts, leaf results
  |-- Irrigation controls
  |-- Agrovet map (OpenStreetMap)
```

## Quick Start

### Prerequisites

- Python 3.9 or higher
- Node.js 18 or higher

### 1. Clone the repository

```bash
git clone <repository-url>
cd smart_farm
```

### 2. Run the start script

```bash
chmod +x start.sh
./start.sh
```

The script handles everything automatically:

1. Checks Python and Node.js versions
2. Creates a Python virtual environment
3. Installs all Python dependencies
4. Installs all npm packages
5. Builds the React frontend
6. Checks for the TFLite model (converts from `.h5` if needed)
7. Starts the Flask server

### 3. Open the app

Go to **<http://localhost:5000>** in your browser. Create an account on the landing page, then sign in to access the dashboard.

Development mode starts Flask and the Vite dev server simultaneously. The Vite server provides hot module replacement so UI changes appear instantly without a full rebuild.

## Simulating Field Data

If you do not have the ESP32 hardware yet, use the built-in simulator to send realistic sensor readings and synthetic leaf images to the backend.

**Run alongside the server:**

```bash
./start.sh --demo                # Normal cycling simulation
./start.sh --demo=dry            # Soil drops until irrigation triggers
./start.sh --demo=hot            # High temperature alert
./start.sh --demo=humid          # Low humidity alert
./start.sh --demo=all            # All alerts at once with diseased leaf images
./start.sh --demo=disease        # Normal sensors, diseased leaf images
./start.sh --demo=healthy        # Normal sensors, healthy leaf images
```

**Run in a separate terminal while the server is already running:**

```bash
cd backend
source venv/bin/activate
python demo_sender.py --scenario dry
```

## ESP32 Firmware Setup

### 1. Install Arduino Libraries

Open Arduino IDE, go to **Sketch > Include Library > Manage Libraries**, and install:

- DHT sensor library (by Adafruit)
- Adafruit Unified Sensor
- ArduinoJson (by Benoit Blanchon)

### 2. Install the ESP32 board package

Go to **File > Preferences** and add this URL to Additional boards manager URLs:

```text
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

Then go to **Tools > Board > Boards Manager** and install **esp32 by Espressif Systems**.

### 3. Configure the firmware

Open `esp32/smart_farm_esp32/smart_farm_esp32.ino` and update these values:

```cpp
const char* WIFI_SSID     = "YOUR_WIFI_SSID";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
const char* SERVER_IP     = "192.168.1.100";  // IP of the machine running Flask
const int   SERVER_PORT   = 5000;
```

To find the IP address of your machine:

```bash
hostname -I       # Linux
ipconfig          # Windows
```

### 4. Flash the firmware

1. Select **AI Thinker ESP32-CAM** as the board in Arduino IDE
2. Set the upload speed to **115200**
3. Hold the **IO0** button on the ESP32-CAM, then press **Reset** to enter flash mode
4. Click **Upload** in Arduino IDE
5. After the upload completes, press **Reset** to start the firmware

## Training the AI Model

A pre-trained TFLite model is included in `model/saved_model/`. If you want to retrain it:

### 1. Download the dataset

Download the PlantVillage dataset from Kaggle:
<https://www.kaggle.com/datasets/emmarex/plantdisease>

Extract it so the class folders (e.g. `Tomato_Early_Blight`, `Tomato_Healthy`) are inside `model/dataset/`.

### 2. Install training dependencies

```bash
cd model
pip install -r requirements.txt
```

### 3. Train

```bash
python train.py
```

Training uses MobileNetV2 with transfer learning and saves:

- `saved_model/best_model.h5` - full Keras model checkpoint
- `saved_model/tomato_disease.tflite` - quantized model for inference
- `saved_model/labels.json` - class index mapping

### 4. Test a prediction

```bash
python predict.py path/to/leaf_image.jpg
```

## API Reference

All endpoints are prefixed with `/api/`.

### Authentication

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | `/api/auth/register` | Create a new account |
| POST | `/api/auth/login` | Sign in |
| POST | `/api/auth/logout` | Sign out |
| GET | `/api/auth/me` | Get current user (401 if not signed in) |

**Request body for register:**

```json
{
  "name": "Jane Farmer",
  "email": "jane@farm.com",
  "password": "yourpassword"
}
```

**Request body for login:**

```json
{
  "email": "jane@farm.com",
  "password": "yourpassword"
}
```

### Sensor Endpoints (used by ESP32)

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | `/api/sensors` | Submit a sensor reading |
| POST | `/api/leaf-image` | Submit a JPEG for disease detection |

**Sensor reading body:**

```json
{
  "temperature": 28.5,
  "humidity": 65.2,
  "soil_moisture": 45
}
```

**Response** includes an irrigation command the ESP32 should act on:

```json
{
  "status": "ok",
  "irrigate": true,
  "alerts": ["Soil moisture low (22%) - irrigation activated"]
}
```

**Leaf image:** send raw JPEG bytes with `Content-Type: image/jpeg`, or multipart form-data with field name `image`.

### Dashboard Endpoints (used by frontend, require login)

| Method | Endpoint | Description |
| --- | --- | --- |
| GET | `/api/status` | Latest sensor values, leaf result, and alerts |
| GET | `/api/history` | Last 200 sensor readings |
| POST | `/api/irrigation/toggle` | Manual irrigation control |

**Irrigation toggle body:**

```json
{ "on": true, "auto": false }
```

Set `auto: true` to return to automatic mode based on soil moisture.

## Configuration

All backend settings live in `backend/config.py`:

```python
# Sensor alert thresholds
SOIL_MOISTURE_LOW   = 30    # % - irrigate when below this
SOIL_MOISTURE_HIGH  = 60    # % - sufficient when above this
TEMP_ALERT_HIGH     = 35    # degrees C
HUMIDITY_ALERT_LOW  = 40    # %

# History
MAX_HISTORY_RECORDS = 200   # readings kept in memory

# Server
HOST  = "0.0.0.0"
PORT  = 5000
DEBUG = False
```

**Setting a secure secret key for production:**

```bash
export SECRET_KEY="your-long-random-secret-key"
./start.sh
```

If `SECRET_KEY` is not set, a default development key is used. Always set it in production.
